00035  * \section Purpose
00036  *
00037  * Implement driver functions for CAN operations.
00038  *
00039  * \section cand_usage Usage
00040  *
00041  * Uses following functions for CAN operations.
00042  * -# Uses CAND_Handler() as peripheral interrupt handler.
00043  * -# Uses CAND_Init() to initialize the driver and peripheral.
00044  * -# Uses CAND_Activate() to enable the CAN interface. Then check
00045  *    CAND_IsReady() to find when CAN is synchronized for data transfer.
00046  * -# To send a CAN message you can:
00047  *    -# Initialize mailbox with CAND_ResetMailbox()
00048  *    -# Start data transfer with CAND_Transfer()
00049  * -# You can also start CAN message by:
00050  *    -# Initialize transfer with CAND_ConfigureTransfer(). Several
00051  *       transfers linked to different mailbox can be initialized.
00052  *    -# After configuration, transfers can be enabled by
00053  *       CAND_StartTransfers() at one time.
00054  * -# To check if the transfer is done, CAND_IsTransferDone() is used.
00055  *
00056  * The following structs should be instanced for driver operations.
00057  * -# \ref sCand : Driver instance struct.
00058  * -# \ref sCandMbCfg : Mailbox configuration parameters list.
00059  * -# \ref sCandTransfer : Message transfer operation parameters list.
00060  *
00061  */
