Detailed Description

Purpose

This directory provides definitions, structs and functions for a USB Composite device demos, to implement Multi-Function USB devices.

Contents

There are three things for the implement of the composite device driver:

  • Implement the function driver structs and functions for the device function, to initialize, to handle function specific requests and dispach standard requests in USBD callbacks, to read/write through assigned USB endpoints.
  • Implement the composite driver structs for the device, to initialize, to handle device specific requests and dispach standard requests in USBD callbacks, to read/write through assigned USB endpoints,
  • Create the composite device's descriptors that should be passed to the USBDDriver instance on initialization, so that the host can recognize the device as a "composite" device.

For more information, please refer to USB Composite Devices Drivers.
