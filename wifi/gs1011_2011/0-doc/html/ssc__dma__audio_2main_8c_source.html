00030 /**
00031  * \page ssc_dma_audio SSC with DMA Audio Example
00032  *
00033  * \section Purpose
00034  * 
00035  * This example uses the Synchronous Serial Controller (SSC) of an AT91SAM9xx5
00036  * microcontroller to output an audio stream through the on-board WM8731 CODEC.
00037  *
00038  * \section Requirements
00039  *
00040  * This package can be used with AT91SAM9xx5 evaluation kits with external 
00041  * codec WM8731 components.
00042  *
00043  * \section Description
00044  * This program plays a WAV file pre-loaded into the flash. The audio stream is
00045  * sent through the SSC interface connected to the on-board WM8731, enabling 
00046  * the sound to be audible using a pair of headphones.
00047  *
00048  * Since the WM8731 DAC requires that it should be feeded a master clock multiple
00049  * of the sample rate, it is difficult to handle any WAV file. As such, this 
00050  * example application is limited to playing files with the following format:
00051  * - Format: WAV
00052  * - Sample rate: 48 kHz
00053  *
00054  * The code can be roughly broken down as follows:
00055  * <ul>
00056  *   <li> Enable the clock </li>
00057  *   <li> Load WAV file information</li>
00058  *   <li> Configure and enable the Codec</li>
00059  *   <li> Configure and enable the SSC interrupt</li>
00060  *   <li> Play WAV file</li>
00061  * </ul>
00062  *
00063  * \section Usage
00064  * -# Build the program and download it inside the evaluation board. Please
00065  *    refer to the
00066  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">
00067  *    SAM-BA User Guide</a>, the
00068  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00069  *    GNU-Based Software Development</a> application note or to the
00070  *    <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00071  *    IAR EWARM User Guide</a>, depending on your chosen solution.
00072  * -# On the computer, open and configure a terminal application
00073  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00074  *   - 115200 bauds
00075  *   - 8 bits of data
00076  *   - No parity
00077  *   - 1 stop bit
00078  *   - No flow control
00079  * -# Start the application.
00080  * -# In the terminal window, the following text should appear:
00081  *    \code
00082  *     -- ssc_dma_audio example xxx --
00083  *     -- AT91xxxxxx-xx
00084  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00085  *     Menu :
00086  *     ------
00087  *     W: Play the WAV file pre-loaded in flash
00088  *     I: Display the information of the WAV file
00089  *     X: Receive WAV file through DBGU
00090  *    \endcode
00091  * The user can then choose any of the available options to perform the described action.
00092  *
00093  * \section References
00094  * - ssc_dma_audio/main.c
00095  * - ssc.c
00096  * - twi.c
00097  * - twid.c
00098  * - dmac.c
00099  * - dmad.c
00100  */
