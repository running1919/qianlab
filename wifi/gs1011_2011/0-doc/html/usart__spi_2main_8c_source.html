00029 /**
00030  *  \page usart_spi  USART SPI Mode Example
00031  *
00032  *  \section Purpose
00033  *
00034  *  This example demonstrates the SPI mode provided by the USART peripherals on
00035  *  AT91SAM9xx5.
00036  *
00037  *  \section Requirements
00038  *
00039  *  This example can be used on AT91SAM9xx5 evaluation kit and usart0 and spi1 pins
00040  *  should be connected as following matching table:
00041  *  - <b> USART0 -- SPI1 </b>
00042  *   - SCK0(PA4) -- SPCK(PA23)
00043  *   - TXD0(PA0) -- MOSI(PA22)
00044  *   - RXD0(PA1) -- MISO(PA21)
00045  *   - RTS0(PA2) -- NPCS0(PA8)  (for USART0 as SPI master)
00046  *
00047  *  - <b> USART0 -- SPI1 </b>
00048  *   - SCK0(PA4) -- SPCK(PA23)
00049  *   - RXD0(PA1) -- MOSI(PA22)
00050  *   - TXD0(PA0) -- MISO(PA21)
00051  *   - CTS0(PA3) -- NPCS0(PA8)  (for USART0 as SPI slave)
00052  *
00053  *  \section Description
00054  *
00055  * This example demonstrates how to use USART in SPI mode. The USART0 is
00056  * configured as SPI master or slave. Meanwhile, the SPI1 peripheral in the
00057  * microcontroller is configured respectively, making it communicate with the
00058  * USART peripheral. The example leaves DBGU to let user to switch the command
00059  * between READ and WRITE from the view of master.
00060 
00061  * The application first initializes DBGU as the interface to interact with
00062  * users. By default, it initializes USART as SPI master and the SPI1 peripheral
00063  * as SPI slave.
00064 
00065  * The application waits for input from DBGU:
00066 
00067  *     - If 'w' is received, the master initiate a transmission by sending a
00068  *     block of data. The slave receives the data and dumps it to DBGU.
00069  *     - If 'r' is received, the master wait for data from slave and dump it to
00070  *     DBGU until the reception is finished.
00071  *     - If 's' is received, the master and slave are exchanged between USART0
00072  *     and SPI1. To assure the transferring succeeds, connect pins as the table
00073  *     above.
00074  *
00075  * \section Usage
00076  *
00077  *  -# Build the program and download it inside the evaluation board. Please
00078  *     refer to the <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">SAM-BA User Guide</a>,
00079  *     the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">GNU-Based Software Development</a>
00080  *     application note or to the <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">IAR EWARM User Guide</a>,
00081  *     depending on your chosen solution.
00082  *  -# Connect a serial cable to the DBGU port on the evaluation kit.
00083  *  -# On the computer, open and configure a terminal application (e.g.
00084  *     HyperTerminal on Microsoft Windows) with these settings:
00085  *        - 115200 bauds
00086  *        - 8 data bits
00087  *        - No parity
00088  *        - 1 stop bit
00089  *        - No flow control
00090  *  -# Start the application. The following traces shall appear on the terminal:
00091  *     \code
00092  *     -- USART SPI Mode Example xxx --
00093  *     -- AT91xxxxxx-xx
00094  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00095  *     \endcode
00096  *
00097  *  \section References
00098  *  - usart_spi/main.c
00099  *  - pio.h
00100  *  - usart.h
00101  */
