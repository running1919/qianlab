The SAM9X25 Software Package will guide you through the best programming usage. Hefty examples of applications can be found here.

Get Started

The SAM9X25 is a ARM926EJ-S based processor. This processor is designed to achieve high system performance in power and cost-sensitive embedded applications. It is also designed to be fast and easy to program, no assembler code or deep knowledge of the architecture to create simple application are required. The Getting Started example provides a simple template to start a new project on SAM9X25.

Microcontroller features

SAM9X25 Microcontroller has many peripheral controllers useful to interact directly with its environment.

Chip Library

Chip Library library consists of a set of functions, data structures and macros covering all peripherals. The development of each driver is driven by a common API (application programming interface) which standardizes the driver structure, the functions and the names of parameters.

  • The Power Management Controller (PMC) optimizes power consumption by controlling all system and user peripheral clocks. The PMC enables/disables the clock inputs to many of the peripherals and the Core. PMC Clock Switching Example gives an example of how to switch from a clock to another (PLLA, PLLUTMI, 32K SLCK/internal RC, MCK) or change divider. System Controller Example allows to measure the consumption of the core in different modes (idle mode, slow clock mode, backup mode), and how to wake-up from back-up mode with RTC alarm or WKUP pin.
  • ADC can convert analog inputs to digital format. adc10 shows how to use adc using several trigger mode with/without DMA. The ADC Controller also includes a Resistive Touchscreen Controller. It supports 4-wire and 5-wiretechnologies. adc_touchscreen shows how to use TouchScreen mode to measure position or pressure and detect pen.
  • TWI interconnects components on a unique two-wire bus, TWI EEPROM Example demonstrates how to use the TWI peripheral to access an external serial EEPROM. TWI Slave Example demonstrates the TWI peripheral in slave mode. It mimics the behavior of a serial memory, enabling the TWI master to read and write data in its internal SRAM. Working with EMAC
  • The Timer Counter "TC" includes three identical 16-bit Timer Counter channels, Each channel can be independently programmed to perform frequency measurement, event counting, interval measurement, pulse generation. TC Capture Waveform Example indicate how to use TC in capture mode to measure the pulse width and count the total pulse number of an external signal injected on TIOA pin.
  • PWM can generate output waveforms and configure it's characteristics. PWM Example make two leds blink thanks to the PWM.
  • WDT can be used to prevent system lock-up if the software becomes trapped in a deadlock. Watchdog with IRQ Interrupt Example configures the watchdog and trigger a interrupt(or reset).
  • The peripherals's DMA operation will done by their own drivers DMAC transfers data from a source peripheral to a destination peripheral over one or more AMBA buses.
  • HSMCI supports the MultiMedia Card (MMC) Specification V4.3, the SD Memory Card Specification V2.0, the SDIO V2.0 specification and CE-ATA V1.1. The Basic SD/MMC Card Example gives deatils on how to implement SD drivers through HSMCI interface.
  • PIO Alternate Function Example gives an example how to alternate ERASE pin between system I/O line and PIO line. microcontroller has the function to alternate some system dedicated pin between PIOs, which extend PIO from existing general purpose I/O.
  • PIO Parallel Capture Example demonstrates how to read data from a CMOS digital image sensor, a high-speed parallel ADC, a DSP synchronous port in synchronous mode, etc.... using the PIO Controller.
  • The SMC is capable of handling several types of external memory and peripheral devices. How to configure SMC to access a LCD and drawing text, image, basic shapes on LCD is detailed in the smc_lcd.
  • SPI is a synchronous serial data link capable of full-duplex communication with external devices in master or slave mode.
  • SSC provides a synchronous communication link with external devices, ssc_pdc_audio describes the configuration required to connect the SSC to a device with an I2S-compatible serial bus.
  • USART provides one full duplex universal synchronous asynchronous serial link. The usart synchronous mode usage is demonstrated in USART Synchronous Mode Example. The USART can works in SPI mode, USART SPI Mode Example would show the example of reading and writing as master or slave. The usart asynchronous mode is shown in the USART serial example, and hardware flow control implementation is illustrated in USART hardware handshaking Example.
  • UDP is compliant with the Universal Serial Bus (USB) V2.0 full-speed device specification. An example on how to implement USB Mass Storage class is detailed in USB Device Mass Storage Example. The USB CDC serial converter shows how to implement USB Communication Device class based on the UDP peripheral. USB Audio Speaker Example details the implementation on USB Audio Device class.

Board Library

Board Library library includes a description of the board releated device drivers plus some source code corresponding to components driver. The firmware library allows any device to be used in the user application without the need for in-depth study of each peripheral specifications.

  • The DMA driver provides DMA interface that can be invoked by the API. The driver use the routines to perform Direct Memory Access (DMA) operations. There are several peripherals (SPI , USART, HSMCI, SSC , ADC, TWI) that can be handled to transfer between these peripherials and memory. The programming of a device's DMA controller is hardware specific, which is descripted is DMAC.

USB Library

USB Library provides a library of highly re-usable usb driver which can be integrated in user applications.

Nand Flash Library

Nand Flash Library package is composed of low-level functions for NAND Flash devices and external API functions that support the unified interface of functions and operations for all supported NAND Flash Devices. Data transfer between NAND flash memory and external memory whether using DMA or Processor, DMA driver performs DMA operations. Nand flash driver takes ECC and Bad Block marking into account, it also support hardware ECC using PMECC module which provided by PMECC Library. The SMC NAND Flash Example show how to read and write data from/to a nandflash connected to the SMC, The nandflash_applet serve as a monitor to implement Nand flash program operations for SAM-BA.

PMECC Library

PMECC Library provides software interface to implement pmecc initialization and some software intensive decodeing algorithm which help to simplity the process of hardware ECC encoding/decoding.

SPI Flash Library

User could access on an external dataflash or serial flash component through SPI. SPI Flash Library combines SPI Interface which provide low level SPI interface shows the usagage of the common spi framework. A Direct Memory Access Controller (DMAC) can also be used to reduce processor overhead, DMA driver performs spi transfer and receive with DMAC. SPI with Serialflash Example are used in the example design to access Atmel SPI flash memory.

Accessing memory through SMC

Static Memory Controller (SMC) is capable of handling several types of external memory and peripheral devices, such as SRAM, PSRAM, PROM, EPROM, EEPROM, LCD Module, NOR Flash and NAND Flash.
The SMC PSRAM Example describes how to configure the Static Memory Controller for PSRAM, and do write and read operations to check the configuration. File-systems and USB mass-storage applications may use the "NAND FTL library" to perform safe access to the NAND in term of power safe, wear leveling and bad block management. The SMC NAND Flash Example show how to read and write data from/to a nandflash connected to the SMC, taking ECC and Bad Block marking into account.The FATFS Nandflash Example will give details to implement filesystem through a NAND FLASH based memory. smc_norflash gives you CFI supported Norflash programming through the External Bus Interfac (EBI).
