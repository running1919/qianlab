00064 /**
00065  *  \page sdmmc_identify SD/MMC Initialization and Identification
00066  * 
00067  *  \section Purpose
00068  *  
00069  *  sdcard sd mode initialization and identification sequence
00070  * 
00071  *  \section sdmmc_identify_desc Description
00072  *  - Host write bit 3 of function 0, register 6 to reset SDIO card (if there is, using Cmd52).
00073  *  - Host sends Cmd0 to do card reset, card is in "idle state".
00074  *  - Host sends Cmd8 and checks the response of the card, only sdcard supports physical
00075  *     layer version 2.00 will response correctly to Cmd8, and this command is mandatory to be
00076  *     sent before ACmd41 for sdcard which support physical layer version 2.00, to enable new
00077  *     functions or to validate a dual-voltage card.
00078  *        - The response result is saved for future branch after "Host sends Cmd5..".
00079  *        - refer to \ref sd_cmd8_ok "If Cmd8 response ok" branch for the initialize of sdcard 2.0.
00080  *        - refer to \ref sd_cmd8_fail "If Cmd8 response fail" branch for the initialize of sdcard 1.x, mmc card, sdcard2.0
00081  *           with invalid voltage.
00082  *  - Host sends Cmd5 with argument 0 and checks the response of the card, only SDIO card supports this command.
00083  *     It starts SDIO initialize. The card returns number of functions in response.
00084  *        - refer to \ref sd_cmd5_ok "If Cmd5 response ok" branch for the initialize of SDIO card.
00085  *  - \anchor sd_cmd5_ok If Cmd5 response ok
00086  *        - Host sends Cmd5 with argument "voltage range" to power up the SDIO card.
00087  *        - If the response failed, it means the card does not match the voltage desired
00088  *          by the host, SDIO type saved as false.
00089  *        - If the response ok, SDIO type saved as true, continue to next step.
00090  *  - \anchor sd_cmd8_ok If Cmd8 response ok
00091  *        - Host sends ACmd41* with argument "HCS" equal to "1". 
00092  *        - If the response to ACmd41 failed, it means the card does not match the voltage 
00093  *           desired by the host, SD card type saved as true.
00094  *        - If the response with "CCS" equal to "1", the card is a version 2.0 high capacity sdcard, 
00095  *           refer to \ref sd_card_init "Card Initialize" for the succeeding initialize sequence.
00096  *        - If the response with "CCS" equal to "0", the card is a version 2.0 standard capacity sdcard.
00097  *           refer to \ref sd_card_init "Card Initialize" for the succeeding initialize sequence.
00098  *  - \anchor sd_cmd8_fail If Cmd8 response fail
00099  *        - Host sends ACmd41* argument "HCS" equal to "0".
00100  *        - If the response to ACmd41 ok, the card is a version 1.x sdcard, refer to \ref sd_card_init "Card Initialize" for 
00101  *           the succeeding initialize sequence.
00102  *        - If the response to ACmd41 fails
00103  *           - If SDIO type is true, refer to \ref sd_card_init "Card Initialize".
00104  *           - Host sends Cmd0 to reset card.
00105  *           - Host sends Cmd1 with argument "hdSupport" equal to "1" to card.
00106  *           - If card has response to Cmd1, the card is a MMC card, refer to \ref sd_card_init "Card Initialize" for the 
00107  *              succeeding initialize sequence. Furthermore, if the response with bit[30:29] equal to 
00108  *              "00" or "11", the card is a High Density MMC, else the card is a standard MMC.
00109  *           - If card has no response to Cmd1, the card is either an unknown card or a card does 
00110  *              not match host's voltage, the initialize ends.
00111  *  - \anchor sd_card_init Card Initialize
00112  *        - If SD or MMC type is true, Host sends Cmd2 to get the its unique card identification number (CID).
00113  *        - Host sends Cmd3 to ask the card to publish a new relative card address (RCA), once the 
00114  *           RCA is received the card state changes to the "stand-by state".
00115  *        - If SD or MMC type is true, Host sends Cmd9 to get the Card Specific Data (CSD).
00116  *        - At this stage, the initialization and identification process is over, the following steps are done 
00117  *           for the sdcard's succeeding operation.
00118  *        - SD(HC) or MMC(HD) memory cards
00119  *           - Host sends Cmd13 to obtain the card status, make sure the card is "ready-for-data".
00120  *           - Host sends Cmd7 to transit card in "transfer state".
00121  *           - If card is a sdcard*, hosts send ACmd6 to set bus to 4-wire mode.
00122  *           - If card is a mmc card, the bus is set as 1-wire mode.
00123  *        - SDIO card
00124  *           - the bus is set as 1-wire mode.
00125  * 
00126  *      \note Send Cmd55 before send ACmd41.
00127  *      \note sdcard include ver 1.x sdcard, ver2.0 standard capacity sdcard, ver2.0 high capacity sdcard
00128  */

00130 /**
00131  *  \page sdmmc_write_op SD/MMC Write
00132  * 
00133  *  \section Purpose
00134  *  
00135  *  sdcard sd mode write process
00136  * 
00137  *  \section Description
00138  *  - Make sure sdcard is under "transfer state", if the sdcard is under other state, host will send
00139  *     Cmd12 to stop the current operation and to transit sdcard to "stand-by state".
00140  *  - Host sends Cmd13 to check sdcard's status, to make sure sdcard is "ready-for-data".
00141  *  - Host sends Cmd25 to do multiple blocks write, the address here is different between high capacity
00142  *     sdcard and normal sdcard, the address of SDHC is equal to the block number, while normal sdcard's
00143  *     address is equal to  block number times 512.
00144  * 
00145  *  \section wr_optimize Write Optimization
00146  *  - To optimize the write multiple blocks, try to keep the sdcard in the "rcv state" as long as possible.
00147  *     after send WRITE_MULTIPLE_BLOCK command, set the block number in the MCI_BLKR as 0, each time a 
00148  *     new write start, do not re-send the WRITE_MULTIPLE_BLOCK command, just re-configure the PDC. In 
00149  *     this case, host does not have to send STOP_TRANSMISSION to transfer the sdcard's state and the
00150  *     performance is optimized.
00151  *  - MoveToTransferState will check the state of the sdcard, and send STOP_TRANSMISSION if need
00152  *    to transfer state. Normally this function is called between the state tranfer.
00153  *  - ContinuousWrite will be called if WRITE_MULTIPLE_BLOCK already been sent and this function
00154  *     will not re-send the write command, but will re-configure PDC accordingly.
00155  */

00204  *
