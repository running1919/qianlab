00030 /**
00031  * \page twi_slave TWI Slave Example
00032  *
00033  * \section Purpose
00034  *
00035  * This project demonstrates the TWI peripheral in slave mode. It mimics the
00036  * behavior of a serial memory, enabling the TWI master to read and write
00037  * data in its internal SRAM.
00038  *
00039  * \section Requirements
00040  *
00041  * This package can be used with AT91SAM9xx5 evaluation kits.
00042  *
00043  * In addition, another device will be needed to act as the TWI master. The
00044  * twi_eeprom example can be used for that, in which case a second kit
00045  * supported by that project is needed.
00046  * -# Connect TWD0 (SDA) for the 2 boards: PA30 of chipset socket pin.
00047  * -# Connect TWCK0 (SCL) for the 2 boards: PA31 of chipset socket pin.
00048  * -# Connect GND for the 2 boards: GND on extension header.
00049  * -# Add a pull up of 2,2KOhms on TWD and TWCK (if not pulled up on customer board)
00050  *
00051  * \section Description
00052  *
00053  * After launching the program, the device will act as a simple TWI-enabled
00054  * serial memory containing 512 bytes. This enables this project to be used
00055  * with the basic-twi-eeprom-project as the master.
00056  *
00057  * To write in the memory, the TWI master must address the device first, then
00058  * send two bytes containing the memory address to access. Additional bytes are
00059  * treated as the data to write.
00060  *
00061  * Reading is done in the same fashion, except that after receiving the memory
00062  * address, the device will start outputting data until a STOP condition is
00063  * sent by the master.
00064  * The default address for the TWI slave is fixed to 0x53. If the board has a TWI
00065  * component with this adress, you can change the define AT24C_ADDRESS in main.c
00066  * of twi_eeprom example, and the define SLAVE_ADDRESS in main.c of twi_slave
00067  * exmaple.
00068  *
00069  * \section Usage
00070  *
00071  * -# Build the program and download it inside the evaluation board. Please
00072  *    refer to the
00073  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">
00074  *    SAM-BA User Guide</a>, the
00075  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">

00078  *    IAR EWARM User Guide</a>, depending on your chosen solution.
00079  * -# On the computer, open and configure a terminal application
00080  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00081  *   - 115200 bauds
00082  *   - 8 bits of data
00083  *   - No parity
00084  *   - 1 stop bit
00085  *   - No flow control
00086  * -# Start the application.
00087  * -# Upon startup, the application will output the following line on the DBGU:
00088  *    \code
00089  *     -- twi slave example xxx --
00090  *     -- xxxxxx-xx
00091  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00092  *     -I- Configuring the TWI in slave mode
00093  *    \endcode
00094  * -# For the TWI Master board, see the description inside his project
00095  * -# and the "Master" board will output:
00096  *    \code
00097  *     -- twi eeprom example xxx --
00098  *     -- xxxxxx-xx
00099  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00100  *     -I- Filling page #0 with zeroes ...
00101  *     -I- Filling page #1 with zeroes ...
00102  *     -I- Read/write on page #0 (polling mode)
