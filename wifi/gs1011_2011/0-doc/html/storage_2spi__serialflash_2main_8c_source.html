00030 /**
00031  * \page spi_serialflash SPI with Serialflash Example
00032  *
00033  * \section Purpose
00034  *
00035  * This example demonstrates how to setup the SPI in order to initialize, read
00036  * and write a serial dataflash.
00037  *
00038  * \section Requirements
00039  *
00040  * This package can be used with AT91SAM9xx5 evaluation kits.
00041  *
00042  * \section Description
00043  *
00044  * The demonstration program tests the serial dataflash present on the
00045  * evaluation kit by erasing and writing each one of its pages. It also gives
00046  * read/write bandwidth by the test.
00047  *
00048  * \section Usage
00049  *
00050  * -# Build the program and download it inside the evaluation board. Please refer to the
00051  * <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6132.pdf">
00052  * SAM-BA User Guide</a>, the
00053  * <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6132.pdf">
00054  * GNU-Based Software Development</a> application note or to the
00055  * <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6132.pdf">
00056  * IAR EWARM User Guide</a>, depending on your chosen solution.
00057  * -# Optionally, on the computer, open and configure a terminal application
00058  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00059  *   - 115200 bauds
00060  *   - 8 bits of data
00061  *   - No parity
00062  *   - 1 stop bit
00063  *   - No flow control
00064  * -# Start the application.
00065  * -# Upon startup, the application will output the following lines on the terminal window:
00066  *    \code
00067  *    -- SPI with Serialflash Example xxx --
00068  *    -- AT91xxxxxx-xx
00069  *    -- Compiled: xxx xx xxxx xx:xx:xx --
00070  *    SPI and AT25 drivers initialized
00071  *    \endcode
00072  * -# The program will connect to the serial firmware dataflash through the SPI
00073  *    and start sending commands to it. It will perform the following:
00074  *    - Read the JEDEC identifier of the device to autodetect it
00075  *      The next line should indicate if the serial dataflash has been
00076  *      correctly identified. For example, this is what appears when an
00077  *      AT25DF321 chip is recognized:
00078  *      \code
00079  *      AT25DF321 Serial Flash detected
00080  *      \endcode
00081  *    - Unprotected the serial flash
00082  *    - Erase the serial flash
00083  *    - Test read bandwidth
00084  *    - Check that each page is blank
00085  *    - Write a "Walking one" pattern on each page:
00086  *      \code
00087  *      Byte 0 = 00000001
00088  *      Byte 1 = 00000010
00089  *      Byte 2 = 00000100
00090  *      ........
00091  *      \endcode
00092  *      Then caculate write bandwidth.
00093  *    - Verify that the pattern has been correctly applied on each page
00094  *    - If all test are passed, the following message will appear:
00095  *      \code
00096  *      Test successfuly achieved.
00097  *      \endcode
00098  *
00099  * \section References
00100  * - spi_serialflash/main.c
00101  * - spi.c
00102  * - spid_dma.c
00103  * - at25_spi.c
00104  * - at25d.c
00105  */
