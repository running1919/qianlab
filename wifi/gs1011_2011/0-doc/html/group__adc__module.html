Working with ADC
[Chip Library, Touchscreen Driver]

Collaboration diagram for Working with ADC:

Purpose

Interface for the Analog-to-Digital Converter (ADC) controller.

It converts the analog input to digital format. The converted result could be 8bit or 10bit. The ADC supports up to 12 analog lines.

Usage

To Enable a ADC conversion,the user has to follow these few steps:

  • Select an appropriate reference voltage on ADVREF
  • Configure the ADC according to its requirements and special needs,which could be broken down into several parts:
    1. Select the resolution by setting or clearing ADC_MR_LOWRES bit in ADC_MR (Mode Register)
    2. Set ADC clock by setting ADC_MR_PRESCAL bits in ADC_MR, the clock is calculated with ADCClock = MCK / ( (PRESCAL+1) * 2 )
    3. Set Startup Time,Tracking Clock cycles and Transfer Clock respectively in ADC_MR.
  • Start conversion by setting ADC_CR_START in ADC_CR.

To use the module for ADC operations:

  1. Initialize the ADC with ADC_Initialize().
  2. Select the active channel using ADC_EnableChannel()
  3. Start the conversion with ADC_StartConversion()
  4. Wait the end of the conversion by polling status with ADC_GetStatus()
  5. Finally, get the converted data using ADC_GetConvertedData()
Note:
PINs are switched to ADC function automatically if the ADC channel is used.

To support Touchscreen monitoring:

  1. Configure ADC as described above.
  2. Enable touchscreen mode with ADC_SetTsMode().
  3. Enable touchscreen pen detection by ADC_SetTsPenDetect(), setup its debounce time with ADC_SetTsDebounce().
  4. Configure touchscreen trigger with ADC_SetTriggerMode().
  5. When pen detected, the ADC samples are reported, X,Y,Pressure values can be read by ADC_GetTsXPosition(), ADC_GetTsYPosition(), ADC_GetTsPressure().
  6. Touchscreen interrupts and status are also managed by ADC functions as ADC_EnableIt(), ADC_DisableIt(), ADC_GetStatus().

For more accurate information, please look at the ADC section of the Datasheet.
