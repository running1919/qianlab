00006 \section sec_getStarted Get Started
00007 The SAM9X25 is a ARM926EJ-S based processor. This processor is designed to achieve high system performance in power and cost-sensitive embedded applications.  It is also designed to be fast and easy to program, no assembler code or deep knowledge of the architecture to create simple application are required. The \ref getting_started "Getting Started" example provides a simple template to start a new project on SAM9X25.

00009 \section microcontroller_sec Microcontroller features
00010 SAM9X25 Microcontroller has many peripheral controllers useful to interact directly with its environment.\n
00011 
00012 \section libChip_sec Chip Library 
00013 \ref lib_chip library consists of a set of functions, data structures and macros covering all peripherals.
00014  The development of each driver is driven by a common API (application programming interface)
00015 which standardizes the driver structure, the functions and the names of parameters.
00016 
00017 - The Power Management Controller (PMC) optimizes power consumption by controlling all system and user peripheral clocks. The PMC enables/disables the clock inputs to many of the peripherals and the Core.
00018 \ref pmc_clock_switching gives an example of how to switch from a clock to another (PLLA, PLLUTMI, 32K SLCK/internal RC, MCK) or change divider. \ref sysc allows to measure the consumption of the core in different modes (idle mode, slow clock mode, backup mode), and how to wake-up from back-up mode with \ref rtc_module "RTC" alarm or WKUP pin.
00019 
00020 - \ref adc_module "ADC" can convert analog inputs to digital format. \ref adc10 shows how to use adc using several trigger mode with/without DMA. The ADC Controller also includes a Resistive Touchscreen Controller. It supports 4-wire and 5-wiretechnologies. \ref adc_touchscreen shows how to use TouchScreen mode to measure position or pressure and detect pen.
00021 - \ref twi_module "TWI" interconnects components on a unique two-wire bus, \ref twi_eeprom demonstrates how to use the TWI peripheral to access an external serial EEPROM.
00022 \ref twi_slave demonstrates the TWI peripheral in slave mode. It mimics the behavior of a serial memory, enabling the TWI master to read and write data in its internal SRAM.
00023 \ref emac_module
00024 - The Timer Counter "TC" includes three identical 16-bit Timer Counter channels, Each channel can be independently programmed to perform
00025 frequency measurement, event counting, interval measurement, pulse generation. \ref tc_capture_waveform indicate how to use TC in capture mode to measure the pulse width and count the total pulse number of an external signal injected on TIOA pin.
00026 - \ref pwm_module "PWM" can generate output waveforms and configure it's characteristics. \ref pwm make two leds blink thanks to the PWM.
00027 - \ref wdt_module "WDT" can be used to prevent system lock-up if the software becomes trapped in a deadlock. \ref wdg_irq "Watchdog with IRQ Interrupt Example" configures the watchdog and trigger a interrupt(or reset).
00028 
00029 - The peripherals's DMA operation will done by their own drivers \ref dmac_module "DMAC" transfers 
00030 data from a source peripheral to a destination peripheral over one or more AMBA buses.
00031 
00032 - \ref hsmci_module "HSMCI"  supports the MultiMedia Card (MMC) Specification V4.3, the SD Memory Card Specification V2.0, the SDIO V2.0 specification and CE-ATA V1.1. The \ref hsmci_sdcard "Basic SD/MMC Card Example"  gives deatils on how to implement SD drivers through HSMCI interface.
00033 - \ref pio_alternate_function "PIO Alternate Function Example" gives an example how to alternate ERASE pin between system I/O line and PIO line.\nThe microcontroller has the function to alternate some system dedicated pin between PIOs, which extend PIO from existing general purpose I/O.
00034 - \ref pio_capture_module "PIO Parallel Capture Example" demonstrates how to read data from a CMOS digital image sensor, a high-speed parallel ADC, a DSP synchronous port in synchronous mode, etc.... using the PIO Controller.
00035 
00036 - The SMC is capable of handling several types of external memory and peripheral devices. How to configure SMC to access a LCD and drawing text, image, basic shapes on LCD is detailed in the \ref smc_lcd.\n
00037 
00038 - \ref spi_module "SPI" is a synchronous serial data link capable of full-duplex communication with external devices in master or slave mode.
00039 
00040 - \ref ssc_module "SSC" provides a synchronous communication link with external devices, \ref ssc_pdc_audio describes the configuration required to connect the SSC to a device with an I2S-compatible serial bus.
00041 
00042 - \ref usart_module "USART" provides one full duplex universal synchronous asynchronous serial link. The usart synchronous mode usage is demonstrated in \ref usart_synchronous. The USART can works in SPI mode, \ref usart_spi would show the example of reading and writing as master or slave.
00043 The usart asynchronous mode is shown in the \ref usart_serial "USART serial example", and hardware flow control implementation is illustrated in \ref usart_hw_handshaking "USART hardware handshaking Example".
00044 
00045 - \ref usb_lib "UDP" is compliant with the Universal Serial Bus (USB) V2.0 full-speed device specification. An example on how to implement USB Mass Storage class is detailed in \ref usb_massstorage "USB Device Mass Storage Example". The \ref usb_cdc_serial "USB CDC serial converter" shows how to
00046    implement USB Communication Device class based on the UDP peripheral. \ref usb_audio_speaker "USB Audio Speaker Example" details the implementation on USB Audio Device class.

00060 \section nandflash_sec Nand Flash Library 
00061 \ref lib_nandflash package is composed of low-level functions for NAND Flash devices and external API functions that 
00062 support the unified interface of functions and operations for all supported NAND Flash Devices. Data transfer between NAND flash memory and external memory whether using DMA or Processor, \ref dmad_module performs DMA operations.
00063 Nand flash driver takes ECC and Bad Block marking into account, it also support hardware ECC using PMECC module which provided by \ref lib_pmecc.
00064 The \ref smc_nandflash show how to read and write data from/to a nandflash connected to the SMC,
00065 The \ref nandflash_applet serve as a monitor to implement Nand flash program operations for SAM-BA.

00071 \section sec_spi SPI Flash Library
00072 User could access on an external dataflash or serial flash component through SPI.
00073 \ref lib_spiflash combines \ref spi_module "SPI Interface" which provide low level SPI interface shows the usagage of the common spi framework.
00074 A Direct Memory Access Controller (DMAC) can also be used to reduce processor overhead, \ref dmad_module performs spi transfer and receive with DMAC.
00075 \ref spi_serialflash are used in the example design to access Atmel SPI flash memory.

00078     \subsection smc_sec Accessing memory through SMC
00079 Static Memory Controller (SMC) is capable of handling several types of external memory and peripheral devices, such as SRAM, PSRAM, PROM, EPROM, EEPROM, LCD Module, NOR Flash and NAND Flash.\n
00080 The \ref smc_psram describes how to configure the Static Memory Controller for PSRAM, and do write and read operations to check the configuration.
00081 File-systems and USB mass-storage applications may use the "NAND FTL library" to perform safe access to the NAND in term of power safe, wear leveling and bad block management.
00082 The \ref smc_nandflash show how to read and write data from/to a nandflash connected to the SMC, taking ECC and Bad Block marking into account.The \ref fatfs_nandflash "FATFS Nandflash Example"
00083 will give details to implement filesystem through a NAND FLASH based memory.
00084 \ref smc_norflash gives you CFI supported Norflash programming through the External Bus Interfac (EBI).
