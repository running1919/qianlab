00030 /**
00031  *  \page adc_adc10 ADC10 Example
00032  *
00033  *  \section Purpose
00034  *
00035  *  The ADC10 example demonstrates how to use ADC peripheral.
00036  *
00037  *  \section Requirements
00038  *
00039  *  This package can be used with AT91SAM9xx5 evaluation kits. To enable 
00040  *  measurement of the ADC input by some configurations.
00041  *
00042  *  \subsection hw_setup Hardware Connection Setup
00043  *
00044  *  - Close jumper JP14 on 1 and 2, to select 3.3v as ADVREF.
00045  *  - Connect PA8(PIO) and PB18(ADTRG) so external trigger can be simulated.
00046  *  - PB8(AD9), PB11(AD0) and PB13(AD2) are used to sample voltage inputs.
00047  *
00048  *  \section Description
00049  *
00050  *  This application is aimed to demonstrate how to use ADC with/without
00051  *  DMA. The example reserves buffers for transfer.<br>
00052  *
00053  *  When working with DMA, it works as a big size buffer for the ADC
00054  *  peripherals, the data will be stored immediately without interfering
00055  *  with the processor.<br>
00056  *
00057  *  The example will send the converted data to DBGU to indicate the
00058  *  conversion for several time is done.<br>
00059  *  <br>
00060  *  Steps to be implemented:
00061  *  <ul>
00062  *   <li>Initialize ADC with expected parameters.</li>
00063  *   <li>Configure and enable interrupt for ADC.</li>
00064  *   <li>Enable DMA reception.</li>
00065  *   <li>Handle DMA done status in DMA transfer callback when DMA used.</li>
00066  *   <li>Check the last converted channel in ADC interrupt handler if DMA not used.</li>
00067  *  </ul>
00068  *
00069  *  \section Usage
00070  *
00071  *  -# Build the program and download it inside the evaluation board. Please
00072  *     refer to the
00073  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">
00074  *     SAM-BA User Guide</a>, the
00075  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00076  *     GNU-Based Software Development</a>
00077  *     application note or to the
00078  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00079  *     IAR EWARM User Guide</a>,
00080  *     depending on your chosen solution.
00081  *  -# On the computer, open and configure a terminal application
00082  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00083  *    - 115200 bauds
00084  *    - 8 bits of data
00085  *    - No parity
00086  *    - 1 stop bit
00087  *    - No flow control
00088  *  -# In the terminal window, the
00089  *     following text should appear (values depend on the board and chip used):
00090  *     \code
00091  *      -- ADC10 Example xxx --
00092  *      -- AT91xxxxxx-xx
00093  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00094  *     \endcode
00095  *  -# The application will output converted value to hyperterminal.
00096  *  -# If touchscreen is connected, AD0 and AD2 values change when press on
00097  *     the touchscreen.
00098  *  -# Connect AD9 to voltage source to measure the voltage value, such as
00099  *     GND, 3V3, or Pin 3 of JP14 (it's about 3v), etc.
00100  *
00101  *  \section References
00102  *  - adc_adc10/main.c
00103  *  - adc.h
00104  */
