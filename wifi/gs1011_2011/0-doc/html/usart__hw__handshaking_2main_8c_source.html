00029 /**
00030  *  \page usart_hw_handshaking USART Hardware Handshaking Example
00031  *
00032  *  \section Purpose
00033  *
00034  *  This example demonstrates the hardware handshaking mode (i.e. RTS/CTS)
00035  *  provided by the USART peripherals on AT91 microcontrollers. The practical
00036  *  use of hardware handshaking is that it allows to stop transfer on the USART
00037  *  without losing any data in the process. This is very useful for applications
00038  *  that need to program slow memories for example.
00039  *
00040  *  \section Requirements
00041  *
00042  *  This example can be used on AT91SAM9xx5 evaluation kits. It requires a 
00043  *  serial line with hardware control support( TXD and RXD cross over, RTS and 
00044  *  CTS cross over) to connect the board and PC.
00045  *
00046  *  \section Description
00047  *
00048  *  The provided program uses hardware handshaking mode to regulate the data
00049  *  rate of an incoming file transfer. A terminal application, such as
00050  *  hyperterminal, is used to send a text file to the device (without any
00051  *  protocol such as X-modem). The device will enforce the configured
00052  *  bytes per second (bps) rate with its Request To Send (RTS) line.
00053  *
00054  *  Whenever the data rate meet or exceed the configurable threshold, the device
00055  *  stops receiving data on the USART. Since no buffer is provided to the DMA,
00056  *  this will set the RTS line, telling the computer to stop sending data. Each
00057  *  second, the current data rate and total number of bytes received are
00058  *  displayed; the transfer is also restarted.
00059  *
00060  *  Note that the device may receive slightly less bytes than the actual file
00061  *  size, depending on the nature of the file. This does NOT mean that bytes
00062  *  have been lost: this is simply an issue with how line breaks are transmitted
00063  *  by the terminal. It is therefore better to use binary files, as they most
00064  *  often do not contain line breaks. For example, send one of the object files
00065  *  generated by the compiler.
00066  *
00067  *  \section Usage
00068  *
00069  *  -# Build the program and download it inside the evaluation board. Please
00070  *     refer to the <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">SAM-BA User Guide</a>,
00071  *     the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">GNU-Based Software Development</a>
00072  *     application note or to the <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">IAR EWARM User Guide</a>,
00073  *     depending on your chosen solution.
00074  *  -# Connect a serial cable to the USART0 port on the evaluation kit.
00075  *  -# On the computer, open and configure a terminal application (e.g.
00076  *     HyperTerminal on Microsoft Windows) with these settings:
00077  *        - 115200 bauds
00078  *        - 8 data bits
00079  *        - No parity
00080  *        - 1 stop bit
00081  *        - Hardware flow control (RTS/CTS)
00082  *  -# Start the application. The following traces shall appear on the terminal:
00083  *     \code
00084  *     -- USART Hardware Handshaking Example xxx --
00085  *     -- AT91xxxxxx-xx
00086  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00087  *     Bps:    0; Tot:      0
00088  *     \endcode
00089  *  -# Send a file in text format to the device. On HyperTerminal, this is done
00090  *     by selecting "Transfer -> Send Text File" (this does not prevent you from
00091  *     sending binary files). The transfer will start and the device will update
00092  *     the bps and total counts on the terminal.
00093  *  -# Whenever the transfer is complete, the total number of bytes received
00094  *     should match the size of the sent file (unless it is a text file, see
00095  *     explanation in description section).
00096  *
00097  *  \section References
00098  *  - usart_hw_handshaking/main.c
00099  *  - pio.h
00100  *  - tc.h
00101  *  - usart.h
00102  */
