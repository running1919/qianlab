00029 /** \cond usb_massstorage
00030  * \page usb_massstorage USB Device Mass Storage Example
00031  *
00032  * \section Purpose
00033  *
00034  * The USB Massstorage Example will help you to get familiar with the
00035  * USB Device Port(UDP) on AT91SAM microcontrollers. Also
00036  * it can help you to be familiar with the USB Framework that is used for
00037  * rapid development of USB-compliant class drivers such as USB Mass
00038  * Storage class (MSD).
00039  *
00040  * \section Requirements
00041  *
00042  * This package can be used with all Atmel evaluation kits that have USB interface
00043  *
00044  * \section Description
00045  *
00046  * The demo simulates a 10M bytes USB disk.
00047  *
00048  * When an EK running this program connected to a host (PC for example), with
00049  * USB cable, the EK appears as a USB Disk for the host. Then the host can
00050  * format/read/write on the disk.
00051  *
00052  * If there is SDRAM on the EK, the disk can be up to 10M so that read/write
00053  * speed can be tested.
00054  *
00055  * If there is no SDRAM but only internal flash, the disk is about 30K and
00056  * only small file can be tested.
00057  *
00058  * \section Usage
00059  *
00060  * -# Build the program and download it inside the evaluation board. Please
00061  *    refer to the
00062  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/6421B.pdf">
00063  *    SAM-BA User Guide</a>, the
00064  *    <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00065  *    GNU-Based Software Development</a> application note or to the
00066  *    <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00067  *    IAR EWARM User Guide</a>, depending on your chosen solution.
00068  * -# On the computer, open and configure a terminal application
00069  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00070  *   - 115200 bauds
00071  *   - 8 bits of data
00072  *   - No parity
00073  *   - 1 stop bit
00074  *   - No flow control
00075  * -# Start the application.
00076  * -# In the terminal window, the following text should appear:
00077  *     \code
00078  *     -- USB Device Mass Storage Example xxx --
00079  *     -- AT91xxxxxx-xx
00080  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00081  *     \endcode
00082  * -# When connecting USB cable to windows, the host
00083  *    reports a new USB %device attachment and Disk installation.
00084  *  . Then new "USB Mass Storage Device" and
00085  *    "ATMEL Mass Storage MSD USB Device" and "Generic volume" appear in
00086  *    hardware %device list.
00087  * -# You can find the new disk on host, and to create/write file to it.
00088  *
00089  * \section References
00090  * - usb_massstorage/main.c
00091  * - pio.h
00092  * - pio_it.h

00094  * - usb: USB Framework, USB MSD driver and UDP interface driver
00095  *    - \ref usbd_framework
